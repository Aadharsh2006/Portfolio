~/about  //  01

Where design judgment meets engineering discipline.

Most developers treat the interface as the last step. I treat it as a first‑class requirement. My work sits at the intersection of structured backend engineering and deliberate interface design — because a system that's technically correct but hard to use hasn't actually solved the problem yet.

That dual lens shapes how I build. When I design a screen, I'm already thinking about the data model behind it. When I write a Django view, I'm thinking about the person who'll eventually click the button it serves. I care about role‑based access that's genuinely secure, inventory logic that doesn't break at the edges, and layouts that don't make the user think twice.

I learn by building real, working systems rather than tutorials — which is why most of my projects start from an actual operational problem, get scoped properly, and end up shipped, not abandoned.

~/work  //  03

// project preview
Full‑stack · Django

Mini Workshop Management ERP System

Problem

Small automotive workshops typically run on paper job cards, spreadsheets, and memory — leading to lost job history, inaccurate spare parts stock, and billing that doesn't match what was actually used or charged.

Solution

A Django‑based ERP that centralizes the full workshop workflow: customer records, digital job cards, spare parts inventory tracking, and itemized billing — gated behind role‑based access so staff, mechanics, and admins each see exactly what their role needs, nothing more.

Tech Stack
Python Django MySQL HTML5 CSS3 JavaScript
Core capabilities
  • Customer & vehicle profile management
  • Job card creation, tracking, and status updates
  • Spare parts inventory with stock‑level visibility
  • Itemized billing generated directly from job cards
  • Role‑based access control for staff, mechanics, and admins
